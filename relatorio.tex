\section*{1. Visão Geral}

O \textbf{TUIpeRacer2} é um jogo de corrida de digitação em Java/JavaFX onde jogadores competem digitando um texto o mais rápido e corretamente possível. Suporta três tipos de jogador: \textbf{Local} (humano), \textbf{CPU} (IA com WPM configurável) e \textbf{TCP} (remoto via rede). A arquitetura segue o padrão \textbf{MVC} com camadas de modelo, serviço, persistência, rede e view.

\section*{3. Hierarquia de Classes}

\textbf{\texttt{Player} (abstrata, implementa \texttt{Typeable})} — raiz da hierarquia. Centraliza atributos comuns: texto a digitar, texto digitado, WPM, acurácia e timestamps. Define \texttt{processInput(char)} como abstrato.

\begin{itemize}[leftmargin=*, itemsep=1pt, topsep=2pt]
    \item \texttt{PlayerLocal} — jogador humano; valida o caractere digitado e lança \texttt{InvalidInputException} em caso de erro.
    \item \texttt{PlayerCpu} — IA com WPM-alvo e jitter; auto-digita via \texttt{autoTick(long nowMs)}.
    \item \texttt{PlayerTcp} — espelho de jogador remoto; recebe caracteres da rede.
\end{itemize}

\textbf{\texttt{Race}} — encapsula estado da corrida (\texttt{WAITING / RUNNING / FINISHED}) e a lista de jogadores.
\textbf{\texttt{RaceResult}} (implementa \texttt{Persistable}) — agrega resultados individuais (\texttt{PlayerResult}) e suporta serialização.\\
\textbf{\texttt{AppContext}} (\emph{Singleton}) — container de serviços: \texttt{RaceService}, \texttt{LobbyService}, \texttt{RaceResultManager}, \texttt{RaceServer}, \texttt{RaceClient}.

\section*{4. Interfaces}

\begin{multicols}{2}
\textbf{\texttt{Typeable}} — contrato de digitação, implementado por toda a hierarquia \texttt{Player}:
\begin{lstlisting}[style=javastyle]
interface Typeable {
  void processInput(char c)
      throws InvalidInputException;
  boolean isFinished();
  double getProgress(); // 0.0-1.0
  double getWPM();
  double getAccuracy(); // 0-100
}
\end{lstlisting}

\columnbreak
\textbf{\texttt{Persistable}} — contrato de serialização, implementado por \texttt{RaceResult} e \texttt{RaceResultManager}:
\begin{lstlisting}[style=javastyle]
interface Persistable {
  String serialize();
  void deserialize(String data)
      throws FileLoadException;
}
\end{lstlisting}
\end{multicols}

\section*{6. Fluxo de Exceção entre Camadas}

\textbf{Cenário:} usuário inicia corrida sem definir texto. A exceção percorre três camadas:

\begin{enumerate}[leftmargin=*, itemsep=2pt, topsep=2pt]
    \item \textbf{Model} — \texttt{Race.start()} detecta texto vazio e lança \texttt{InvalidTextException}.
    \item \textbf{Service} — \texttt{RaceService.startRace()} propaga a exceção (declarada em assinatura).
    \item \textbf{View} — \texttt{SetupController} captura e exibe \texttt{Alert} ao usuário:
\end{enumerate}

\begin{lstlisting}[style=javastyle]
// Race.java (Model)
public void start() throws InvalidTextException {
    if (text == null || text.isBlank())
        throw new InvalidTextException("Texto nao pode ser vazio.");
    state = RaceState.RUNNING;
}

// SetupController.java (View)
try {
    raceService.startRace();          // Service propaga
    navigateToRace();
} catch (InvalidTextException e) {
    new Alert(Alert.AlertType.ERROR, "Defina um texto para a corrida.").show();
}
\end{lstlisting}

\section*{7. Coleções Utilizadas}

\begin{table}[H]
\centering\small
\renewcommand{\arraystretch}{1.15}
\begin{tabularx}{\textwidth}{llX}
\toprule
\textbf{Coleção} & \textbf{Local} & \textbf{Finalidade} \\
\midrule
\texttt{ArrayList<Player>}                      & \texttt{Race}            & Lista polimórfica de jogadores \\
\texttt{ArrayList<PlayerResult>}                & \texttt{RaceResult}      & Resultados individuais da corrida \\
\texttt{ArrayList<RaceResult>}                  & \texttt{RaceResultManager} & Histórico persistido em arquivo \\
\texttt{ArrayList<LobbyEntry>}                  & \texttt{LobbyService}    & Lobbies online disponíveis \\
\texttt{ConcurrentHashMap<Integer,ClientHandler>} & \texttt{RaceServer}    & Clientes TCP thread-safe \\
\texttt{HashMap<String,PlayerPane>}             & \texttt{RaceController}  & Lookup de painel UI por nome \\
\texttt{ConcurrentLinkedQueue<RemoteChar>}      & \texttt{RaceController}  & Fila não-bloqueante de eventos de rede \\
\bottomrule
\end{tabularx}
\end{table}

O \texttt{ArrayList<Player>} em \texttt{Race} é central: permite iterar polimorficamente sobre todos os tipos de jogador. O \texttt{ConcurrentLinkedQueue} desacopla a thread de rede do \texttt{AnimationTimer} JavaFX sem bloqueios.
